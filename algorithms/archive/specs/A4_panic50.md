# A4_panic50

C1_final 베이스 + panic threshold 25% → 50% 완화.

## 동기

C1_final의 단점: 단기 거래 부족 (1주 0건, 2주 3건).
Panic 50%로 완화해서 단기 거래 빈도 증가 시도.

## 룰

C1_final과 동일하지만:
- **Panic threshold: 25% → 50%**

```
[진입]
  - 어제 일봉 close 후 (KST 09:00) 평가 → 오늘 시초가 매수
  - 어제 low ≤ (10일 최저가 × 1.01)         ← low 1%
  - 어제 일봉 양봉 (close > open)
  - 동시 보유 < 5종
  - 가드 통과
  → 자본 20% 매수
  - 종목 풀: 40개

[청산]
  - 매일 일봉 close 평가
  - 익절: +5% / 손절: -2.5% / 5일 강제
  - 다음날 시초가 매도

[가드]
  - BTC 일봉 trend uptrend일 때만 매수, 아니면 즉시 청산
  - 40종 중 50%+ 동시 신저가면 매수 X + 청산   ← panic 25% → 50%
  - 자본 -10% drawdown 도달 시 모두 청산 + 10일 휴식

[재진입]
  - 같은 종목 청산 후 24h cooldown
```

## 인사이트

- panic threshold 25% → 50%로 약간 완화해도 알파 유지
- panic 30~35%는 오히려 손해 (false trigger 잡음)
- panic 50%는 진짜 폭락만 잡음 → 적절한 방어 + 거래 빈도 ↑
- 라비 우려 (단기 거래 부족)에 가장 적합
